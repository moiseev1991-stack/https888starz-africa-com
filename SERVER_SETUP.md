# Настройка на сервере (Beget / 888starzeg-egypt.com)

## Если `/assets/js/app.js` отдаёт 500 Internal Server Error

### 1. Права на файлы

На скриншотах у файлов атрибуты **600** (чтение/запись только владелец). Веб-сервер должен иметь право читать файлы.

**Сделай на хостинге (файловый менеджер или FTP):**

- Файлы (в т.ч. `app.js`, `.css`, `.html`): права **644**
- Папки (`assets`, `js`, `apk` и т.д.): права **755**

Иначе Apache не сможет прочитать `app.js` и может отдать 500 или 403.

### 2. .htaccess в корне сайта

В репозитории в `public/.htaccess` добавлены правила: **существующие файлы отдаются как статика**, без передачи в PHP. После деплоя этот файл должен лежать в корне сайта (рядом с `index.html`, `assets/`).

Если на Beget в корне уже есть свой `.htaccess` (например, от другой CMS), в нём не должно быть правил вида «все запросы перенаправлять в index.php» без проверки «если файл существует — не перенаправлять». Иначе запрос к `app.js` уйдёт в PHP и даст 500.

### 3. После изменений

- Повторно задеплой из GitHub (или залей обновлённый `public/` по FTP).
- В Cloudflare: Caching → Purge Everything.
- Проверь в браузере: `https://888starzeg-egypt.com/assets/js/app.js` — должна открыться JS-страница, а не страница с ошибкой.
