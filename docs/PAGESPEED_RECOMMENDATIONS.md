# Рекомендации PageSpeed (по аудиту)

Краткий список действий по скринам и отчётам.

- **A) TypeError: $ is not a function**  
  jQuery не загружен до скрипта или `$` перезаписан. Уже частично решено (перенос инлайн-скриптов после jQuery в optimize-html-performance.js). При необходимости: подключать jQuery до scripts.min.js, использовать `jQuery(function($){ ... })`, или переписать на vanilla JS.

- **B) 404 на /cdn-cgi/rum**  
  Cloudflare RUM/beacon. В HTML/оптимизаторе ссылки на cdn-cgi/rum удаляются. Если 404 остаётся — скрипт инжектится Cloudflare; отключить в панели CF или не трогать.

- **C) Цепочка критических запросов**  
  defer для некритичных скриптов уже стоит. Дополнительно: разнести критичный/некритичный JS, грузить некритичный позже; критичный CSS — первым, остальной — отложенно.

- **D) Кэширование**  
  В config/.htaccess задано 1 год для статики. Если в отчёте всё ещё «7 д.» — кеш переопределяется на Beget; проверить панель/nginx.

- **E) Кнопки карусели без названий**  
  Реализовано: в footer.php при инициализации Owl Carousel для каждого `.owl-dot` задаётся `aria-label="Slide N"` и `aria-current="true"` у активной точки; при смене слайда (onChanged) aria-current обновляется.

- **F) В списках не только &lt;li&gt;**  
  Внутри `<ul class="ez-answer">` не должно быть прямых детей кроме `<li>`. Кнопку «اعرض المزيد» обернуть в `<li>...</li>`.

- **G) Низкое разрешение изображений**  
  Иконки/лого 32×32 при ожидаемых 48×48 (retina). Подготовить размеры под device pixel ratio (например 96×96 для отображения 48×48).
