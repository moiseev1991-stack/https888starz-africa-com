# Optimization Report — статический сайт

Оптимизацию выполнять **после** достижения визуальной идентичности статики с оригиналом.

## 5.1 Удаление лишнего

- Отключить/удалить неиспользуемые CSS/JS в финальной статике (без поломки верстки).
- Убрать wp-emoji, embed-скрипты, лишние preload/prefetch при наличии.

## 5.2 Сжатие и минификация

- Минификация CSS/JS (например esbuild, clean-css, terser).
- Cache-busting: hash в именах файлов или рекомендации по Cache-Control (см. nginx.conf.snippet).

## 5.3 Изображения

- Проверить самые тяжёлые изображения.
- WebP/AVIF + fallback где допустимо; width/height и lazyload без ухудшения вида.

## 5.4 Шрифты

- Google Fonts → по возможности self-host woff2.
- Preload для основных woff2 (умеренно).

## 5.5 Серверные рекомендации

- Файл: `config/nginx.conf.snippet` — Cache-Control, gzip, immutable для статики.
- Аналогичные заголовки для Apache — в .htaccess.snippet при необходимости.

## Что не делалось и почему

- Агрессивное удаление CSS/JS без построения графа зависимостей — риск поломки верстки; делать выборочно после тестов.
- Полная замена всех изображений на WebP без fallback — возможны проблемы в старых браузерах; внедрять с проверкой.

## Lighthouse / Performance

- Цель: Performance 90+ для главной после оптимизаций.
- Основные факторы: размер изображений, шрифты, блокирующие скрипты; описать в отчёте по результатам замеров.
